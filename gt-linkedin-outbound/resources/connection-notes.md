# Connection Notes - The 300-Character Decision

You decide whether to send a connection note, and if yes, what it says. The connection note is LinkedIn's closest equivalent to an email subject line - it's the only piece of copy the prospect sees BEFORE deciding to accept. Treated wrong, it lowers acceptance. Treated right, it lifts it.

## The Default: NO NOTE

Across most cold outbound campaigns, no-note connection requests achieve **equal or higher** acceptance rates than noted ones.

| Setup | Average acceptance |
|---|---|
| No note | 28-32% |
| Generic note ("Hi, I'd love to connect") | 14-18% |
| Specific signal-based note | 30-40% |
| Pitchy note ("I help companies like yours...") | 8-12% |

**A weak note actively reduces acceptance** because it triggers the "this is a salesperson" filter. With no note, the prospect evaluates your profile - which (if optimized) is the strongest signal you have.

## Send a Note ONLY When

| Condition | Why |
|---|---|
| Strong specific signal (recent post, job change <30 days, mutual event) | The signal earns the message; without it, generic notes hurt |
| Mutual connection worth name-dropping (with permission) | Social proof boosts acceptance significantly |
| Senior persona (VP+) where a note signals respect | Skipping a note to a CEO can read as careless |
| Unusual / non-obvious value prop | Profile alone won't communicate it |
| Engagement-first warming was done (you liked or commented their post) | Note can reference the engagement naturally |

For everything else → just hit Connect.

## The 300-Character Hard Constraint

LinkedIn enforces 300 characters max on free-tier accounts. Sales Nav users get the same. There's no workaround.

**Optimal length: 150-220 characters** (about 25-40 words). This is short enough to feel human, long enough to say something meaningful.

Going over the limit at all = forced truncation. Test EVERY note in the actual LinkedIn UI before scaling.

## Rules for All Connection Notes

- **No subject line** - DMs/notes don't have one
- **No "Hi {{firstName}}, I came across your profile"** - most-burned phrase, instant decline
- **No "I'd love to add you to my professional network"** - LinkedIn's default copy, signals laziness
- **No company name (yours)** - gives away the sales motive
- **No CTA, calendar link, or "got 15 min?"** - the note is for getting connected, NOT for selling
- **No emojis** - 🚀 ✨ 🎯 are instant declines in B2B outbound
- **No flattery** - "Loved your post" / "Really impressive" - burns trust
- **One specific reference** - not their job title, something earned
- **"You" > "I"** - always
- **Read aloud test** - must sound like a peer, not a vendor

## 6 Connection Note Patterns

### 1. Recent Post Reference (strongest)
```
Saw your post on [topic] - [one-sentence genuine reaction].
Worth being connected.
```

**Why it works:** Proves you read their content. References something they're proud of (they wrote it). Creates social currency.

---

### 2. Job Change Recognition
```
Saw the move to [Company] - that transition usually means [specific challenge].
Following along.
```

**Why it works:** New role = recipient is in change-mode. They appreciate someone noticing without pitching.

---

### 3. Mutual Context (event, group, school)
```
We were both at [event/group] - didn't get to chat.
Building out my [function] network in [industry].
```

**Why it works:** Mutual context = de-risks the connection. They classify you as "person from that thing" rather than "salesperson."

---

### 4. Industry Peer
```
Building things in [industry] - your work at [Company] is on my radar.
Open to connecting?
```

**Why it works:** Positions you as a peer with parallel work. Lower threat.

---

### 5. Specific Hire / Team Signal
```
Saw [Company] is scaling [team]. That phase is its own kind of chaos.
Worth being connected.
```

**Why it works:** Trigger-based, references something they're actively living, no pitch.

---

### 6. Curious Frame (use sparingly - only when no signal exists)
```
Building a network of [role]s working through [common challenge].
You came up - open to connecting?
```

**Why it works:** Honest framing. Lower-converting than signal-based notes but better than nothing if you must send a note.

## Anti-Patterns (What Kills Acceptance)

```
❌ "Hi {{firstName}}, I'd love to connect with you on LinkedIn!"
❌ "Hi {{firstName}}, I came across your profile and was impressed."
❌ "Hi {{firstName}}, I help companies like {{company}} achieve [outcome] - let's connect!"
❌ "Saw you're {{title}} at {{company}} - want to chat?"
❌ "Hey! 👋 Loved your work - let's connect!"
❌ "Hi {{firstName}}, are you the right person to talk to about [thing]?"
❌ "Hi {{firstName}}, I'd love to learn more about what you do at {{company}}."
```

Each averages <15% acceptance. They train LinkedIn's spam classifier to flag your account.

## ATL vs BTL Note Strategy

| Seniority | Default | If sending a note |
|---|---|---|
| **VP / C-Suite** | Note recommended | Brief observation, peer-to-peer tone, signal respect |
| **Director** | Note optional | Use if signal exists |
| **Manager** | No note often outperforms | Only if signal is strong |
| **IC** | No note almost always better | Profile speaks for itself |

The intuition: senior people receive lots of connection requests and use the note as a filter for "this person made an effort" vs "this is automated." Junior people are flattered by being connected with at all and the note often introduces unnecessary risk.

## A/B Testing Connection Notes

Test 2 variations at a time:

| Test type | What to compare |
|---|---|
| Note vs No-note | Run 50% of cold list with no note, 50% with the same note. Measure acceptance + reply rate downstream. |
| Signal-based vs Generic | Same audience, two notes - one references a profile signal, one doesn't |
| Short vs Medium | 100-char note vs 200-char note, same content compressed |
| Question vs Statement | "Worth being connected?" vs "Worth being connected." |

Minimum sample size: 100 connection requests per variant before drawing conclusions.

**Track BOTH:**
- Acceptance rate (the immediate KPI)
- Downstream reply rate to M1 (connections from "high-effort notes" often reply at higher rates)

A note variant with 5% lower acceptance but 2x reply rate is the better choice.

## Examples

### Example 1 - VP Sales, recent post about pipeline visibility

**With note (under 200 chars):**
```
Your post on the gap between forecast accuracy and pipeline hygiene - sharp.
Most VPs fix one or the other, never both. Worth being connected.
```

**No-note alternative:** Just send it. Sender's headline + sender's recent post on RevOps will do the work.

**Recommendation:** Test both at 50/50 split. Senior persona = lean toward note.

---

### Example 2 - Marketing Ops Manager, no specific signal available

**Recommendation:** No note.

**Why:** Manager-level + no signal = generic note will reduce acceptance vs letting the sender's profile speak. If the sender has good content on marketing ops in the feed, acceptance rate will be higher with no note.

---

### Example 3 - Founder, post-Series A funding

**With note:**
```
Saw [Company] just closed [round] - that phase usually means rebuilding GTM 
from scratch. Building out my founder network around [function]. Open to connecting.
```

---

### Example 4 - Director of RevOps, hired 60 days ago

**With note:**
```
Saw the move to [Company] 2 months in. New RevOps role usually means inheriting 
3 tools that don't talk to each other. Following along - worth being connected.
```

---

### Example 5 - IC Engineer, no signal

**Recommendation:** No note.

**Why:** ICs respond worst to noted requests in B2B outreach. The note creates suspicion ("why is a sales person reaching out to me?"). No-note + a strong sender profile (real engineer, real company, real content) outperforms.

## Pre-Send List Hygiene (Lift Acceptance Before Touching Copy)

Before optimizing note vs. no-note, sharpen the list itself. The single highest-impact filter most teams skip:

**Sales Navigator filter: "Posted on LinkedIn in the last 30 days."**

Reaching out to a prospect who hasn't logged in since 2023 is wishful thinking, not outreach. Filtering on recent posting activity surfaces prospects who are *actually using the platform* - they'll see the request, they'll evaluate it, they'll respond. Adding this filter alone tends to lift acceptance immediately, before any copy changes.

Other pre-send hygiene that compounds:

| Filter / signal | Why it lifts acceptance |
|---|---|
| Mutual connections ≥5 | Profiles with shared context get noticeably higher acceptance |
| Recently changed roles (last 90 days) | New-role prospects are evaluating tools and open to peer outreach |
| Engaged with your or your team's content | Some warming has already happened - build on it |
| Posted recently on a relevant topic | Real signal you can reference without feeling like a stalker |

After the request is sent, the next hygiene rule is **withdrawing pending invitations after 14 days.** A queue >1000 pending requests triggers LinkedIn's anti-spam systems and quietly suppresses your sender's reach. Most automation platforms (HeyReach included) auto-withdraw on a configurable schedule - set it to 14 days and forget it.

## Quick Diagnostics

| Symptom | Likely Cause | Fix |
|---|---|---|
| Acceptance <15% | Generic note + weak sender profile | Drop the note + optimize sender profile (headline, banner, recent posts) |
| Acceptance 15-22% | Signal-light note OR weak ICP match | Tighten ICP, OR add a real signal to notes, OR drop notes |
| Acceptance 22-30% | Average performance | Consider engagement-first warming (lift +10pts) |
| Acceptance 30%+ | Strong setup | Don't change anything until you have 500+ data points |
| Reply rate low after acceptance | Note set wrong expectations OR M1 is too pitchy | Diagnose M1 separately (route to dm-sequence) |
