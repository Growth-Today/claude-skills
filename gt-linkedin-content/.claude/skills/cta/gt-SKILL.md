## CTA Performance Data (from 2,759-post dataset)

| CTA Type | Avg Comments | Verdict |
|----------|--------------|---------|
| Comment [keyword] to get [resource] | **267** | Best for BoF — 2.7x baseline |
| Repost ask | 109 | Good for ToF reach plays |
| No CTA | 97 | Fine for short MoF posts |
| Soft pointer (👇⬇️) | 95 | MoF when no resource |
| Follow ask | 88 | ToF audience growth |
| Link CTA | 87 | Rarely — LinkedIn suppresses links |
| Opinion ask ("Agree?") | **57** | **Never use** — worst performing |

**Rule:** One CTA per post. Multiple CTAs dilute action. Never use "Agree?" — it statistically underperforms everything else.

## P.S. — Mandatory in Every Post

P.S. appears in **63% of top-performing posts** across the dataset. It is the #1 structural element by usage frequency. It is not optional.

### The Three P.S. Types

**Type 1 — Resource offer (BoF posts):**
```
P.S. We built the [specific resource] for this workflow.
Comment "[KEYWORD]" and I'll send it over.
Make sure we're connected.
```

**Type 2 — Service signal (pipeline posts):**
```
P.S. We build these systems for B2B GTM teams at Growth Today.
If you want us to set this up for you — DM me "GTM".
```

**Type 3 — Follow CTA (ToF/MoF posts without a resource):**
```
P.S. I break down exactly how we build these systems every week.
Follow Growth Today if you want the next one.
```

### P.S. Rules
- Always on its own line, separated from the body by a blank line
- Never more than 3 lines
- Never put an external link in the P.S. (LinkedIn suppresses external links)
- After P.S., the post is done — no more content
- Match P.S. type to funnel stage: BoF → Type 1 or 2, ToF/MoF → Type 3

## CTA by Funnel Stage

### ToF (Awareness) — maximize reach and reposts
```
[Soft pointer] 👇
P.S. [Follow CTA — Type 3]
```
Or:
```
Repost this if your team needs to see it.
P.S. I break down [topic] every week. Follow Growth Today.
```

### MoF (Consideration) — maximize C/R ratio and discussion
```
Over to you: [Specific question about their situation]
P.S. [Follow CTA or service signal depending on post]
```
Examples:
```
Over to you: what's your current ICP research setup?
Over to you: which part of this workflow would you change first?
Over to you: are you running this manually or automated?
```

### BoF (Conversion) — maximize intent signals
```
Comment "[KEYWORD]" and I'll send it over.
Make sure we're connected so I can DM you.
P.S. [Service signal — Type 2]
```

## Comment-Gate Best Practices

Comment-gate posts get **3.7x more comments** than non-gated posts (GT data). The algorithm sees high comment volume → boosts distribution → more reach.

**Requirements to use comment-gate:**
1. You have a real deliverable to send (template, guide, list, playbook)
2. You are connected to commenters or can DM them
3. You follow up and actually deliver

**Format:**
```
Comment "[KEYWORD]" and I'll send it over.
Make sure we're connected so I can DM you.
```

**Examples of KEYWORD choices:**
- "ICP" → for ICP frameworks
- "CLAY" → for Clay templates
- "STACK" → for tech stack breakdowns
- "SYSTEM" → for workflow reveals
- "GTM" → for GTM playbooks
- "SIGNAL" → for intent signal lists
