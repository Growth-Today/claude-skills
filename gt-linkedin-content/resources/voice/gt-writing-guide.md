## Content Philosophy — The Growth Today DNA

1. **Show the system, not just the result** — walk through exactly how we built it, step by step
2. **Specific numbers always** — every claim has a number. Never "many", never "several", always "37" or "12 hours" or "4.7% reply rate"
3. **Building in public** — process transparency, client results, tool stack, team decisions
4. **Practitioner voice** — we've done this, not just read about it. "We built" not "you should consider building"
5. **Actionable depth** — readers should be able to replicate the system after reading
6. **Visuals multiply reach** — image posts get 2.3x reactions vs text-only. Always recommend a visual.

## Writing Voice & Tone

### Core rule: structured operator voice

GT writes like a practitioner who has built the system and is showing you exactly how it works. Not conversational musing — structured, numbered, result-first.

### DO:

- **Start with the result or bold claim** — never context first
- **Step 1 / Step 2 / Step 3** for any process or walkthrough
- **Arrow lines (↳)** for sub-points under each step — the GTM LinkedIn signature format
- **"We built" / "We tested" / "We ran"** — agency operator voice
- **Specific numbers always**: "657 leads", "12 hours/week", "4.7% reply rate", "40 meetings in 6 weeks"
- **"Here's what we did / Here's the exact system / Here's the workflow"** — standard reveal bridge
- **P.S. section** — mandatory at the end of every post (see P.S. rules below)
- **Bold unicode** for 1-3 key phrases per post (+124% reach)
- **Generous whitespace** — visual break every 2-3 lines, no walls of text (+212% reactions)
- **"We" for agency/team posts, "I" for personal story posts**

### DON'T:

- **Start with an observation hook** — "We ran...", "Last week...", "I was thinking...", "I noticed...", "Recently..." — these are setup sentences, not hooks. This is Brigi's old style and explicitly banned. EVERY post starts with a result, a number, or a contrarian statement. The ONLY exception: personal story hook where the NUMBER is in line 1 (e.g. "I turn 31 today. $557K last month." — number IS the story).

- Start with context, preamble, or "I've been thinking about..."
- Use vague quantifiers: "many", "several", "a lot", "tons of"
- Write walls of text — every 2-3 lines needs a visual break
- Use: leverage, unlock, game-changer, synergy, align, deep dive, revolutionize, paradigm, disrupt, empower, holistic, robust, thought leadership
- Put an external link in the body (40-60% reach penalty)
- Start AND end with a question — pick one
- Use more than 2 emojis per post
- Use "Agree?" as a CTA (57 avg comments — worst performing)
- **Bold, italic, or any markdown formatting in the post text** — LinkedIn plain text only, always. No exceptions. Bold unicode (𝗯𝗼𝗹𝗱) is allowed in visuals/design but NEVER in the written post copy.

## @Tagging Rule — Mandatory on Every Post

Every post must include 2-4 @tags. Never more than 4 — LinkedIn penalizes over-tagging.

**What to tag:**
- Tools used in the workflow described (e.g. @Clay, @Instantly, @HubSpot, @n8n)
- People mentioned by name in the post (e.g. @Jani Vrancsik if co-author)
- Partner companies if directly relevant (e.g. @Clay Enterprise)

**Rules:**
- Tag tools only if they appear in the post content — never force-tag for reach
- 2 tags minimum on every post, 4 maximum
- Tags go in the post body where the tool/person is first mentioned, not at the end
- Never tag competitors
- Never tag someone just for visibility — only if genuinely referenced

## Sentence & Structural Rules

**Median line: 7 words. Hook lines: 9 words. Hard ceiling: 20 words.**
Sweet spot: 4-14 words per line. 75% of all lines in top posts fall here.

**Post length:**
- Target: 250-500 words (avg 392 reactions)
- Never under 100 words (avg 136 reactions — consistently underperforms)
- 500+ words only for deep system reveals or case studies

**Paragraphs:** 1-2 sentences max, then whitespace break. No exceptions.

**Formatting hierarchy:**
1. Step 1: / Step 2: / Step 3: — for any process
2. ↳ — for sub-points (preferred over bullets)
3. → — for action items or flow
4. Bold unicode — 1-3 key phrases, never more
5. Emojis: 0-2 max, used structurally only (👇 before CTA)

## P.S. — Mandatory Structural Element

P.S. appears in 63% of top-performing posts. Every post ends with one of three P.S. types:

**Type 1 — Resource offer (BoF):**
```
P.S. We built the Clay table template for this workflow.
Comment "ICP" and I'll send it over.
Make sure we're connected.
```

**Type 2 — Service signal (pipeline):**
```
P.S. We build these systems for B2B GTM teams at Growth Today.
If you want us to set this up for you — DM me "GTM".
```

**Type 3 — Follow CTA (ToF/MoF):**
```
P.S. I break down exactly how we build these systems every week.
Follow Growth Today if you want the next one.
```

**P.S. rules:**
- Always on its own line, separated from the body
- Never more than 3 lines
- Never put an external link here
- After P.S., the post is done — no more content

## Default Body Structure: The Step Framework

For any process, workflow, or system reveal:

```
[HOOK — bold result or claim, max 15 words]

[1-line credibility or context]

Here's [what we built / the exact system / how it works]:

Step 1: [Action label]
↳ [1-2 sentences of detail]
↳ [1-2 sentences of detail]

Step 2: [Action label]
↳ [1-2 sentences of detail]
↳ [1-2 sentences of detail]

Step 3: [Action label]
↳ [1-2 sentences of detail]

The result: [specific metric or outcome]

[CTA — question, comment-gate, or soft pointer]

P.S. [One of three P.S. types above]
```

## Pre-Publish Checklist

- [ ] First line is a result, bold claim, or number — NOT context
- [ ] Every major claim has a specific number
- [ ] Step 1/2/3 structure used for any process content
- [ ] ↳ arrows for sub-points (not dashes or bullets)
- [ ] Visual break every 2-3 lines — no wall of text
- [ ] 250-500 words target (never under 100)
- [ ] Post ends with P.S. (one of three types)
- [ ] One CTA only
- [ ] No external links in body or P.S.
- [ ] Max 2 emojis total
- [ ] No banned vocabulary (leverage, game-changer, etc.)
- [ ] Bold unicode used for 1-3 key phrases
- [ ] "We" for team posts, "I" for personal stories
- [ ] Visual recommended (image = 2.3x text-only reactions)
- [ ] Would I stop scrolling for this hook?
